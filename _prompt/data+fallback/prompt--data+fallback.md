В папке data/mosmetro-ru есть все, для получения данных с сайта mosmetro.ru для последующего 
автономного построение маршрутов между 2-мя станциями метро и вычисления времени в пути.

В папке data/metrobook-ru - аналогично для получения данных с сайта metrobook.ru.

# Задача

## Получение и храннени исходной информации

Сформируй код, который 1 раз в сутки автоматически скачивает данные с сайта mosmetro.ru 
Если данные с ссайта mosmetro.ru получить не удается, то скачивать данные с сайта metrobook.ru.
и сохраняет их на диск в папку, находящуюся вне контроля версий.


В памяти создается необходимый кеш для расчета вариантов кратчайших маршрутов между 2-мя станциями метро
и ля вычисления времени в пути.

Сведения из mosmetro.ru содержат больше информации, чем metrobook.ru.
В частности, есть сведения о том, из какого вагона удобнее выходить и входить на пересадках, 
какие маршруты общественного транспорта находятся рядом со станцией метро, есть сведения о закрытых станциях
и ремонтах.

В памяти и на диске должен быть такй формат днных, который предусматривает откат к более скудному варианту. 
При работе с более скудным вариантом данных просто будет возвращаться меньше сведений по маршрутам,
или не будут учитываться закрытия и ремонты.

Отметим, что в отличие от сведений о станциях метро, которые могут храниться на диске вечно, пока не доступны оба источника данных,
сведения о закрытиях и ремонтах имеют время жизни 24 часа. 
Поскольку они буду пытаться обноситься один раз в 24 часа, то, при следующем обновлении, если эти данные недоступны с сайта  
mosmetro.ru, файл с диска с устаревшими сведениями должен быть удален.


Что касатеся сведений о станциях метро и тп., если оба сайта недоступны, то использует данные с диска.
Если оба сайта не доступны, и не диске еще нет данных, то кеш пустой. И при попытке рассчитать маршрут, код, использующий данные кеша, будет возвращать ошибку.

Модуль с функциями поиска кратчайших машрутов и рассчета ввремени в пути пометси в папку `src/lib`

Эти функции должны возвращать всю возможную информацию о маршруте:
- станции, 
- пересадки, 
- время на перегонах между пеерсадками
- в какие вогоны входить на пеерсадках
- сведения о маршрутах наземного транспорта на конечных станциях
- сведения о том, чвляется ли линия МЦД
- сведения о ремонтах эскалаторов, входов, лифтов
- и прочие сведения, которые предосталяет mosmetro.ru

## Организация неточного поиска по станциям метро.

После получения полного перечня станций, необходимо их загнать в кеш для неточного поиска.
Функция неточного поиска должна находить набор из N (=5) подходящих станций, отсортированных по убыванию схожести.
Если максимальная схожесть меньше некотрого порога, то возвращется 0 станций.
Если схожесть >= 1, то возвращается одна станция.
В остальных случаях возвращается от 2 до N станций.
