## Что сделано

**Слой данных** (`src/lib/metro-data/`). Код раз в сутки автоматически скачивает данные метро: основной
источник — mosmetro.ru (полная схема плюс уведомления о закрытиях и ремонтах), при его недоступности —
резервный metrobook.ru (граф извлекается прямо из HTML главной страницы). Скачанное сохраняется в папку
`data-cache/` вне контроля версий с атомарной записью и контрольными суммами. Каскад при сбоях: свежий
mosmetro → свежий metrobook → дисковая копия mosmetro → дисковая копия metrobook → пустой кеш с понятной
ошибкой `MetroDataUnavailableError`. Файл уведомлений живёт 24 часа: если обновить их не удалось, файл
удаляется — схема же с диска не удаляется никогда. Планировщик подключён в `start.ts`: при старте сервер
мгновенно поднимает дисковую копию, затем в фоне обновляется из сети.

**Единый формат с деградацией.** Оба источника нормализуются в один тип `IMetroDataset`: обязательное ядро
(станции, линии, рёбра с временем в секундах) и необязательные «богатства» mosmetro (вагоны, наземный
транспорт, закрытия, названия на четырёх языках). При работе от metrobook эти поля просто отсутствуют —
без веток по источнику. Вдобавок скудный набор обогащается из последней сохранённой схемы mosmetro:
подтягиваются иностранные названия и имена станций пересадочных узлов для поиска.

**Маршрутизация** (`src/lib/routing/`). Алгоритм Дейкстры и алгоритм Йена (несколько вариантов маршрута),
применение закрытий на заданный момент времени, включая временные обходные рёбра. Ответ содержит всё
доступное: этапы со списком станций, время каждого участка, в какой вагон садиться на пересадке, признак
МЦД/МЦК, наземный транспорт у конечных станций, время входа/выхода, предупреждения о ремонтах. Эталонная
сверка: «Ховрино → Тёплый Стан» даёт 57/59/60 минут — совпадает с сайтом; закрытая «Серп и Молот»
корректно объезжается по альтернативному ребру и снова открывается после 31.08.2026.

**Неточный поиск** (`src/lib/station-search/`). Алгоритм из mcp-jira, усиленный тремя доработками:
компонент «мешка слов» (находит «стан теплый»), языковая нормализация для арабского (огласовки, варианты
алефа) и китайского, расширение латинских запросов обратной транслитерацией («hovrino» находит «Ховрино»
как точное совпадение). Правила соблюдены: ниже порога — пусто, точное совпадение — только точные
(одноимённые станции разных линий — все, с указанием линии), иначе — от 2 до 5 кандидатов.

**Тесты** — 62 штуки в `tests/lib/` на реальных фикстурах: маршруты, закрытия, все ветки каскада с
подменёнными сетью и диском, срок жизни уведомлений, и широкий прогон поиска с опечатками, склейкой и
перестановкой слов на русском, английском, арабском и китайском. Живая проверка тоже пройдена: слой
данных скачал реальные данные (443 станции, 92 уведомления) в `data-cache/`.

**Отчёт** — `claudedocs/dev-report.md`: решения, покрытие, находки и ограничения.

## Одно замечание, требующее вашего действия

Команда `yarn start` в этой рабочей копии падает не из-за нового кода: шаблону сервера нужен секрет для
админ-панели (`webServer.auth.jwtToken.encryptKey`) в гитигнорируемом файле `config/local.yaml`, которого
здесь нет. Слой данных метро при этом успевает корректно инициализироваться — это видно в журнале запуска.
Чтобы сервер поднялся целиком, создайте `config/local.yaml` с этим ключом (или отключите вкладку jwtToken
в `adminPanel.authType`).
