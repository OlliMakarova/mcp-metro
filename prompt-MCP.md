# Goal
Создать универсальный mcp инструмент, который возвращает либо сведения о станции метро, либо сведения о кратчайших маршрутах
между двумя станциями.

# Instructions
Универсальный инструмент называется Mos_metro_info.
Входные параметры: 
- first_metro_station - название станции отправления или станции, о которой мы хотим получить информацию
  В обработчике инструмента должен осуществляться неточный поиск подходящей станции по переданному значению. 
  Оно может быть передано на любом из 4-х языков с ошибками или опечатками. Если по переданному значению найдена точно 1 станция,
  то она используется. Если найдены несколько вариантов, то ответом инструмента должна быть просьба выбрать подходящую станцию из найденных вариантов.
  Если не найдено ни одной станции, то ответ пользователю - просьба уточнить название станции.
- second_metro_station - название станции прибытия 
  В обработчике инструмента должен осуществляться неточный поиск аналогично first_metro_station. 
  Важно: если переданы обе станции и обе требуют уточнения, то запрашивать уточнение у пользователя сразу по двум станциям.
  Значит в ответе инструмента с просьбой уточнения должно быть 2 списка станций для уточнения.
- action - тип действия, энумератор с двумя значениями: search_route - поиск маршрута, get_station_info - получение информации о станции


В случае search_route, в хэндлере проверять обязательность указания обеих станций.
Возвращать исчерпывающую информацию о маршрутах от 1 до 4-х штук:
- полное время в пути с учетом времени ожидания на станциях
- станции,
- пересадки,
- время на перегонах между пересадками
- в какие вогоны входить на пересадках
- сведения о маршрутах наземного транспорта на конечных станциях
- сведения о том, является ли линия МЦД
- сведения о ремонтах эскалаторов, входов, лифтов
- и прочие сведения, которые предосталяет mosmetro.ru
  

В случае get_station_info, second_metro_station не обязательна. 
Возвращать искерпывающую информацию о станции.

Важно: все ответы инструмента оформлять в markdown. Структуру передавать списками и таблицами.


# Task

- Удалить шаблонные и тестовые инструменты, ресурсы, эндпоинты. 
- Создай новый инструмент.
- Сделай REST эндпоинты:
  - неточный поиск станиции по ее наименованию
  - получение сведений о станции
  - получение маршрутов до 4-х штук с сопутствующими сведениями
  Обязательно предусмотри rate limiting.
- Сформулируй AGENT_BRIEF в `src/prompts/agent-brief.ts` и AGENT_PROMPT в `src/prompts/agent-prompt.ts`
- Вместо тестовых примеров в `tests/mcp/test-cases.js` напишите тесты для нашего случая.
- Обновите README.md, чтобы отразить новый проект. Этот README.md будет использоваться при поиске этого MCP в системе RAG. 
  Поэтому опиши суть инструмента, его возможности, способы установки, список инструментов, 
  ресурсов, подсказки. Кратко и по делу.
