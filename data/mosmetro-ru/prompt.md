На сайте https://www.mosmetro.ru/metro-map
представлена карта моссковского метро.

При загрузке страницы видно получение данных о станциях московского метрополитена https://prodapp.mosmetro.ru/api/schema/v1.0
в JSON есь секции:
- connections
- lines
- stations
- transitions

Сведения о закрытых станциях, ремонтах выходов, эскалаторах, лифтах: https://prodapp.mosmetro.ru/api/notifications/v2

В интерфейсе можно выбрать станции "откуда" и "куда".
И получить варианты маршрутов и время в пути.
(предположительно, эта инфа берется из сведений, полученных из https://prodapp.mosmetro.ru/api/schema/v1.0)

# Задача:

1) Вясни, как считается время в пути между станциями.
2) Как находятся варианты маршрутов между станциями.
2) Придумай методику + программный код, с помощью которого можно минимальным количеством запросов к этому ресурсу получить все сведения для последующего автономного
   использования данных для построения оптимальных маршрутоа между заданными 2-мя станциями и рассчета вреемни в пути.
   С учетом сведений о закрытых станциях и ремонтах.

Код оформляй в виде ванильного JavaScript
Напиши документ MD, где простым языком изложи алгоритмы. Там же приведи свои наблюдения, риски.
