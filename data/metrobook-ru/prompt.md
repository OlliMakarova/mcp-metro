Посмотри https://metrobook.ru/
Можно ли с этого сайта получить всю информацию для рассчета кратчайших быстрейших маршрутов между станциями метро и рассчета времени межу ними, аналогично mosmetro.ru?
Сведения, скрипты, документацию сохрани в папку data/metrobook-ru

----

Готово. Отвечаю на вопрос по существу.

Ответ: да, metrobook.ru полностью пригоден для расчёта маршрутов, и это даже проще, чем с mosmetro.ru

Главная находка: весь взвешенный граф метро зашит прямо в HTML главной страницы — достаточно одного
запроса GET https://metrobook.ru/. Внутри страницы лежит скрипт, который наполняет структуры: вершины
«станция на линии», перегоны с временем хода в секундах (поле ttime), таблица времён пересадок (значение                                                                                                            
999999 означает запрещённый переход) и группировка вершин в физические станции. Названия станций                                                                                               
извлекаются из подписей на схеме в той же странице.

Сам сайт, в отличие от mosmetro.ru, считает маршрут не в браузере, а на сервере — через POST-запрос к
/kcmsajax.php (операция getShortestPath, умеет минимизировать либо время, либо число пересадок). Этот
API я тоже задокументировал — он полезен как эталон для сверки, но для автономной работы не нужен.

Проверка сошлась идеально. Контрольный маршрут «Ховрино → Тёплый Стан»: мой автономный расчёт по данным
metrobook — 57 минут, серверный расчёт самого metrobook — 3420 секунд, то есть те же 57 минут, наш прежний
расчёт по данным mosmetro.ru — тоже 57 минут (сайт mosmetro показывает 56 — разница в округлении).

Что сохранено в data/metrobook-ru:
- fetch-data.js — скачивает страницу, извлекает граф и названия, сохраняет metrobook-graph.json;
- route-cli.js — автономный поиск маршрута алгоритмом Дейкстры по этим данным;
- metrobook-graph.json — нормализованный граф (312 станций, 439 вершин, 421 перегон, 352 пересадки);
- README.md — документация: устройство данных, серверный API, сравнительная таблица с mosmetro.ru, риски;
- index.html, bundle.js, inline-data.js — исходники сайта, по которым делался анализ.

Но есть существенные ограничения по сравнению с mosmetro.ru:
- времена только с точностью до минуты (у mosmetro — до секунды), на длинных маршрутах копится погрешность;
- нет сведений о закрытиях и ремонтах — временно закрытую станцию «Серп и Молот» никак не распознать;
- у пересадочного узла одна подпись: узел «Пушкинская — Тверская — Чеховская» подписан только «Пушкинская»,
  поэтому поиск по названию «Тверская» не сработает (таких «потерянных» названий около 45);
- нет координат, выходов в город, времени входа и выхода, расписаний; покрытие МЦД неполное; встречаются
  опечатки («Малахово» вместо «Малаховка»); сайт любительский, вёрстка может измениться в любой момент.

Рекомендация: основным источником оставить mosmetro.ru (точнее, полнее, есть официальные закрытия с
датами и обходами), а metrobook.ru использовать как независимый резерв и средство перекрёстной проверки —
что мы фактически уже сделали, и результаты совпали. Уникальная особенность metrobook — исторические схемы
метро с 1903 года, если такая задача когда-нибудь возникнет.
