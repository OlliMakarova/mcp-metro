# metrobook.ru как источник данных для расчёта маршрутов метро

Ответ на вопрос: **да, с сайта https://metrobook.ru/ можно получить всю информацию для автономного расчёта
быстрейших маршрутов и времени в пути между станциями** — причём это даже проще, чем у mosmetro.ru: весь
взвешенный граф метро (станции, перегоны с временем хода, пересадки с временем перехода) зашит прямо в HTML
главной страницы, поэтому достаточно **одного запроса** `GET https://metrobook.ru/`.

Проверка подтвердила пригодность данных: автономный расчёт маршрута «Ховрино → Тёплый Стан» по данным
metrobook даёт 57 минут — ровно столько же, сколько наш расчёт по данным mosmetro.ru (57 минут) и сколько
возвращает собственный серверный расчёт metrobook (`totalTime: 3420` секунд = 57 минут). Сам сайт
mosmetro.ru показывает 56 минут — расхождение в одну минуту объясняется округлением.

## Как сайт устроен изнутри

В отличие от mosmetro.ru, который считает маршрут в браузере, metrobook.ru считает маршрут **на сервере**:
при нажатии «построить маршрут» страница отправляет POST-запрос к `/kcmsajax.php` и получает готовый путь.
Но данные для расчёта всё равно публикуются целиком на клиенте — граф нужен странице для отрисовки схемы.

### Структуры данных в инлайн-скрипте главной страницы

Внутри HTML есть скрипт (~64 КБ), наполняющий объект `mb`:

| Структура | Смысл | Пример |
|---|---|---|
| `mb.arrSD[sdid]` | «Станция на линии» — вершина графа: `{sid, lid, sN}` (станция, линия) | `mb.arrSD[460]={sid: 379, lid: 7}` |
| `mb.arrS[sid]` | Физическая станция — группа вершин: `{sdids: [...]}` | `mb.arrS[59]={sdids: [68,69,490,610]}` |
| `mb.arrR[rid]` | Перегон: `{ttime, sdid1, sdid2, lid}`, `ttime` — время хода **в секундах** | `{ttime: 120, sdid1: 67, sdid2: 227}` |
| `mb.arrTT[a][b]` | Время пересадки в секундах между вершинами одного узла; `999999` — переход запрещён | `mb.arrTT[68][69]=180` |
| `mb.arrL[lid]` | Линия: `{type}` — 0 метро, 1 МЦК, 2 МЦД | `mb.arrL[17]={type: 1}` |

Названия станций лежат не в скрипте, а в вёрстке подписей схемы:
`<span mb_sd_id='460' class='stName ...'>Ховрино</span>` — атрибут `mb_sd_id` связывает подпись с вершиной.

В текущей выгрузке: 312 физических станций, 439 вершин «станция × линия», 421 перегон, 352 записи о
пересадках. Граф полностью связный. Все времена кратны минуте (60–660 секунд на перегон).

### Серверный API расчёта маршрута (для сверки)

```
POST https://metrobook.ru/kcmsajax.php
Content-Type: application/x-www-form-urlencoded

mod=metrobook&oper=getShortestPath&sdid1=460&sdid2=202&mid=2&whatToMinimize=0
```

Параметры: `sdid1`, `sdid2` — вершины отправления и назначения; `mid=2` — идентификатор текущей схемы;
`whatToMinimize` — что минимизировать: `0` — время, `1` — число пересадок. Ответ:

```json
{ "success": true,
  "table": { "totalTime": 3420, "transitTime": 3300, "changeTime": 120, "changeNumber": 1,
             "ans": [460, 461, 138, ..., 202] } }
```

`totalTime` — общее время в секундах, `transitTime` — езда, `changeTime` — пересадки, `ans` — цепочка
вершин пути. Авторизация не требуется. Для автономной работы этот API не нужен — он полезен только как
эталон для сверки собственного расчёта.

## Алгоритм автономного расчёта

Тот же, что и для данных mosmetro.ru (подробное объяснение — в `../mosmetro-ru/README.md`):

1. Вершины графа — «станции на линии» (`sdid`), рёбра — перегоны (`ttime` секунд) и пересадки
   (`arrTT` секунд, значения `999999` отбрасываются как запрещённые).
2. Быстрейший маршрут — алгоритм Дейкстры. Поскольку физическая станция — это группа вершин, поиск
   запускается сразу от всех вершин станции отправления до любой вершины станции назначения.
3. Время маршрута = сумма секунд перегонов и пересадок; штраф за пересадку уже зашит в граф.
4. Варианты маршрутов при необходимости — алгоритм Йена (реализован в `../mosmetro-ru/metro-graph.js`).

## Файлы в этой папке

| Файл | Назначение |
|---|---|
| `fetch-data.js` | Скачивает главную страницу, извлекает граф и названия, сохраняет `metrobook-graph.json` |
| `route-cli.js` | Автономный поиск маршрута: `node route-cli.js "Ховрино" "Тёплый Стан"` |
| `metrobook-graph.json` | Нормализованный граф: станции, вершины, перегоны, пересадки |
| `index.html` | Скачанная главная страница (исходник данных, для контроля изменений) |
| `bundle.js`, `inline-data.js` | Рабочие копии скриптов сайта, по которым выполнялся анализ |

Запуск: `node fetch-data.js`, затем `node route-cli.js "Откуда" "Куда"`. Node.js 18+, без зависимостей.

## Сравнение с mosmetro.ru

| Критерий | mosmetro.ru | metrobook.ru |
|---|---|---|
| Запросов для полной выгрузки | 2 (схема + уведомления) | 1 (главная страница) |
| Точность времени перегона | секунды (например, 133 с) | только целые минуты |
| Закрытия станций и ремонты | есть отдельное API с датами и обходами | **нет** (только `999999` у запрещённых переходов) |
| Время входа/выхода на станцию | есть (`enterTime`/`exitTime`) | нет |
| Координаты станций и выходы в город | есть | нет |
| Расписание первых/последних поездов | есть | нет |
| Покрытие | 443 станции, все МЦД до конечных | 312 станций, МЦД частично, без дальних участков |
| Формат данных | чистый JSON API | JavaScript-код и вёрстка внутри HTML (нужен парсинг) |
| Названия станций | у каждой станции каждой линии | одна подпись на пересадочный узел (см. риски) |
| Дополнительно | рекомендации по вагонам, сервисы станций | исторические схемы метро по годам (1903–наши дни) |

**Вывод и рекомендация.** Для боевой задачи расчёта маршрутов основным источником стоит оставить
mosmetro.ru: точность до секунды, официальные сведения о закрытиях с датами и обходными рёбрами, полное
покрытие МЦД. Данные metrobook.ru пригодны как независимый резервный источник и как средство перекрёстной
проверки расчётов (что мы и сделали — результаты совпали). Уникальная ценность metrobook — исторические
схемы метро разных лет, если такая задача когда-нибудь появится.

## Наблюдения и риски

- **Одна подпись на пересадочный узел.** Metrobook хранит узел как одну «физическую станцию», и подпись у
  неё одна: например, узел «Пушкинская — Тверская — Чеховская» подписан только «Пушкинская», поэтому поиск
  по названию «Тверская» ничего не найдёт. Около 45 «вторых» названий узлов отсутствуют. Это главное
  практическое ограничение при поиске станций по имени.
- **Точность — минута.** Все времена кратны 60 секундам; на длинных маршрутах накапливается погрешность
  в несколько минут по сравнению с посекундными данными mosmetro.ru.
- **Нет сведений о закрытиях.** Временные закрытия станций и участков (как «Серп и Молот» летом 2026 года)
  в данных не отражаются — станция просто отсутствует или присутствует без пометок. Для актуальных
  ограничений всё равно нужен запрос уведомлений mosmetro.ru.
- **Данные внутри HTML, а не в API.** Извлечение опирается на текущую вёрстку (`mb.arrSD[...]`,
  `<span mb_sd_id=...>`); любое изменение вёрстки сломает парсер. В `fetch-data.js` стоит проверка с
  понятной ошибкой на этот случай.
- **Есть опечатки.** Например, станция «Малаховка» на схеме подписана «Малахово». Встречаются и другие
  мелкие расхождения названий с официальными.
- **Сайт любительский.** Это авторский проект без гарантий доступности и актуальности; официальный источник
  (mosmetro.ru) обновляется быстрее и надёжнее.
- Значение `999999` в таблице пересадок — соглашение «переход запрещён/закрыт», его нужно отбрасывать при
  построении графа, иначе Дейкстра может выбрать «пересадку» длиной 11 дней.
